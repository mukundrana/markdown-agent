## ⚠️ CRITICAL: TIMESTAMPS AND STAGE NUMBERS

### 🔴 REAL TIMESTAMPS ARE MANDATORY

**❌ FORBIDDEN - Never use these placeholder values:**
```javascript
"timestamp": "ISO-TIMESTAMP"           // ❌ WRONG
"timestamp": "{{ISO_TIMESTAMP}}"       // ❌ WRONG
"timestamp": "TIMESTAMP"               // ❌ WRONG
"timestamp": "2025-03-08T13:26:00.000Z" // ❌ WRONG (all same!)
```

**✅ CORRECT - Use actual current timestamp:**
```javascript
"timestamp": "2026-03-08T13:35:17.452Z"  // ✅ RIGHT - real timestamp
```

**How to generate REAL timestamps:**
```javascript
// Use JavaScript's Date object:
new Date().toISOString()
// Example output: "2026-03-08T13:35:17.452Z"
```

**⚠️ PROGRESSIVE TIMESTAMPS:**
- Each log entry MUST have a DIFFERENT timestamp
- Timestamps should INCREASE throughout the workflow
- For example:
  - log-1: `2026-03-08T13:26:00.000Z`
  - log-2: `2026-03-08T13:26:30.000Z` (+30 seconds)
  - log-3: `2026-03-08T13:27:00.000Z` (+30 seconds)
  - ...and so on
