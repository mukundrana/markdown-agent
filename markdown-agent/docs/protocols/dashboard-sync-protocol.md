# DASHBOARD SYNC PROTOCOL (MANDATORY)

> **CRITICAL**: ALL AGENTS MUST FOLLOW THIS PROTOCOL AFTER EVERY ACTION

---

## 🚨 MANDATORY SYNC AFTER EVERY ACTION

**After EVERY action, EVERY agent MUST:**

1. **Update their task files** (state.json, log.json, checkpoints.json)
2. **SYNC to session/data.js** for dashboard visibility
3. **SYNC to session/queue.json** for queue state (if applicable)

---

## SYNC FUNCTION

```javascript
// CALL THIS AFTER EVERY ACTION
function syncToDashboard(taskId) {
  // 1. READ current state files
  const state = READ_JSON(`session/tasks/${taskId}/state.json`);
  const log = READ_JSON(`session/tasks/${taskId}/log.json`);
  const checkpoints = READ_JSON(`session/tasks/${taskId}/checkpoints.json`);
  const queue = READ_JSON(`session/queue.json`);

  // 2. UPDATE session/data.js
  WRITE_JS(`session/data.js`, `
window.DASHBOARD_DATA = {
  queue: ${JSON.stringify(queue, null, 2)},
  tasks: {
    "${taskId}": {
      state: ${JSON.stringify(state, null, 2)},
      log: ${JSON.stringify(log, null, 2)},
      checkpoints: ${JSON.stringify(checkpoints, null, 2)}
    }
  }
};
  `.trim());

  // 3. UPDATE timestamp in data.js header
  // Last updated: ${new Date().toISOString()}
}
```

## STEP-BY-STEP SYNC PROCESS

### After ANY Action (All Agents)

```
STEP 1: Complete your action
STEP 2: UPDATE session/tasks/task-N/state.json
STEP 3: APPEND to session/tasks/task-N/log.json
STEP 4: CREATE checkpoint in session/tasks/task-N/checkpoints.json (if milestone)
STEP 5: SYNC to session/data.js (MANDATORY!)
STEP 6: SYNC to session/queue.json (if queue state changed)
STEP 7: Continue to next step
```

---

## AGENT-SPECIFIC SYNC REQUIREMENTS

### TASK_MANAGER

After adding/updating tasks:
```
1. UPDATE session/queue.json with new task
2. CREATE session/tasks/task-N/ directory
3. CREATE initial state.json, log.json, checkpoints.json
4. SYNC session/data.js (includes queue + new task)
```

### REQUIREMENTS-GATHERER

After requirements complete:
```
1. UPDATE state.json (stage: PLANNING)
2. APPEND to log.json
3. CREATE checkpoint
4. SYNC session/data.js
```

### PLANNER, CRITIC, SYNTHESIZER

After planning complete:
```
1. UPDATE state.json (stage: IMPLEMENTATION)
2. APPEND to log.json
3. CREATE plan.json
4. CREATE checkpoint
5. SYNC session/data.js
```

### CODER

After each wave:
```
1. UPDATE state.json (currentWave: N)
2. APPEND to log.json with files created/modified
3. SYNC session/data.js
```

### TESTER, REVIEWER, ALL QUALITY CHECKERS

After stage complete:
```
1. UPDATE state.json (next stage number)
2. APPEND to log.json
3. CREATE report file
4. CREATE checkpoint
5. SYNC session/data.js
```

### ORCHESTRATOR

After EVERY stage transition:
```
1. VERIFY state.json was updated by previous agent
2. SYNC session/data.js with latest data
3. TRIGGER next agent
```

After task completion:
```
1. MARK task as completed in state.json
2. UPDATE queue.json (increment completed, decrement inProgress)
3. FIND next pending task
4. If next task exists: START it and sync
5. If no next task: Notify all complete
6. SYNC session/data.js
```

## COMMON SYNC MISTAKES TO AVOID

❌ **FORGOTTEN SYNC**: Updated state.json but didn't sync to data.js
❌ **PARTIAL SYNC**: Synced only state, forgot log or checkpoints
❌ **WRONG TIMESTAMP**: Didn't update "Last updated" timestamp
❌ **STALE DATA**: Read old data.js instead of fresh JSON files
