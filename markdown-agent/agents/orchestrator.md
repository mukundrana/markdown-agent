## Context Management (MANDATORY)

**You are responsible for preventing context window limit errors.**

### Token Monitoring

After each stage, check token usage:

```json
// Check session/tasks/task-N/state.json
{
  "context": {
    "tokenCount": {{CURRENT_TOKEN_COUNT}},
    "compressionLevel": "standard",
    "archivedStages": ["{{PREVIOUS_STAGE}}"]
  }
}
```

### Context Compression Protocol

```
After stage completes:
1. CALL summarizeStage(completedStage)
2. UPDATE state.json.context.summaries[completedStage]
3. CHECK tokenCount vs thresholds:
   - > 80% (120K): Reduce to "minimal" level
   - > 93% (140K): Archive all but current stage
   - > 97% (145K): PAUSE and alert user
4. IF needed: CALL archiveOldLogs()
5. CONTINUE to next agent
```

### summarizeStage() Function

```
Function: Compress completed stage logs to summary
Input: stage name, log entries
Output: summary object with:
  - summary: 1-2 sentence description
  - keyOutcomes: array of key results
  - filesCreated/Modified: array of files
  - duration: time taken
```

### getPrunedContext() Function

```
Function: Return essential context for next agent
Input: currentStage, contextLevel
Output: pruned context object with:
  - summary of completed stages
  - recentLogs: last 3 log entries
  - currentState: state.json snapshot
  - currentPlan: relevant plan excerpt
```

### findNextPendingTask() Function

```
Function: Find next task to execute based on priority
Input: queue.json tasks array
Output: taskId of next pending task

Algorithm:
1. Filter tasks by status = "pending"
2. Sort by priority (critical > high > medium > low)
3. Within same priority, sort by createdAt (FIFO)
4. Return first task's ID
```
